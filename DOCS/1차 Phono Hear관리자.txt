📌 Phono Hear 관리자 수정·추가 내용 정리
(뉴로톡 DTx 기획서 내 Phono Hear 관련 부분 기준)

1. 유저관리
기존 뉴로톡 유저관리 기능과 동일하게 동작.

2. 진짜단어 관리
검색 조건 추가: 언어, 단어, IPA, 소리구분타입(양순음, 치조음, 경구개음, 연구개음, 후음, 받침), 상태(활성/비활성)로 검색 가능.

목록 컬럼 확장:

단어 생성 버튼 → 단일 생성 / 액셀 업로드 생성 2가지 방식 제공.

업로드된 단어 영상파일이 있는 경우 파일명 표시, 클릭 시 해당 접속.

단일 생성/수정 페이지 개선:

활성화 상태(ON/OFF) 설정

소리구분(7종) 지정

언어코드(ko/en/jp/cn)

정답단어 + IPA

변동단어(가짜단어) + IPA

단어 영상파일 업로드

엑셀 업로드 생성 방식 개선:

1행은 제목, 2행부터 데이터 저장

소리구분은 사전 설정된 7종 외 값은 저장 안 함

업로드 시 자동 활성화

영상파일 첨부 불가

3. 미션관리 - 게임진행미션
미션 종류에 진짜단어 유형 추가 (기존 가짜단어만 지원).

진행 기간, 노출 여부, 리워드 포인트 지급 조건 설정 가능.

완료 횟수는 해당 게임 1회 완료 시 1 증가.

💡 요약하면, 이번 Phono Hear 관리자 관련 변경은 단어 관리 고도화(검색·수정·영상 연동·엑셀 업로드 규칙), 미션 종류 확장(진짜단어 추가), 뉴로톡 관리자 기능 일부 재활용이 핵심입니다

📋 구현 순서 및 상세 작업 항목

1️⃣ 진짜단어 관리 완성 (최우선)

A. 소리구분 타입 세분화
- RealWordSearchForm.vue: phonoDivisionOptions 수정
  현재: ['단모음', '장모음', '이중모음', '파열음', '마찰음', '비음', '유음', '반모음', '기타']
  → 요구사항: ['양순음', '치조음', '경구개음', '연구개음', '후음', '받침'] + 추가 분류
- RealWordCreateForm.vue: 동일 옵션 수정
- DataWordsMapper.xml: 검색 조건 업데이트

B. 단어 생성 방식 완성
- RealWordCreateChoicePopup.vue: 단일생성/엑셀업로드 선택 팝업 UI 구현 (현재 빈 파일)
- RealWordListTable.vue: "단어 생성" 버튼을 팝업 호출로 수정
- 백엔드 API: /admin/insertRealWordExcelAjax 엔드포인트 구현
  * 1행 제목, 2행부터 데이터 처리
  * 소리구분 7종 외 값 제외 로직
  * 업로드 시 자동 활성화 처리

C. 영상파일 표시 및 URL 접속 기능
- RealWordListTable.vue: 영상파일 컬럼 추가
- 파일명 표시 및 클릭 시 URL 접속 기능 구현
- videoFileUrl 필드 연동

2️⃣ 미션관리 - 진짜단어 유형 추가

A. Game Constants 확장
- GameConstants.java: 진짜단어 게임 상수 추가
  * GAME_ID_G08 = "G08" (진짜단어)
  * GAME_SUB_ID_G0801~G0807 (소리구분별)

B. 게임 로직 확장
- GameServiceImpl.java: 진짜단어 게임 처리 로직 추가
- DataWordsService: selectRealWordListByGame 메서드 추가
- 미션 완료 조건 설정 (1회 완료 = 1 증가)

C. 미션 관리 UI 확장
- 미션 종류 선택에 진짜단어 옵션 추가
- 진행 기간, 노출 여부, 리워드 포인트 설정 가능

3️⃣ 검색 조건 완성

A. 백엔드 검색 로직 강화
- DataWordsMapper.xml: 새로운 소리구분 타입 검색 조건 추가
- 언어/단어/IPA/소리구분타입/상태별 정확한 필터링

B. 프론트엔드 검색 UI 개선
- RealWordSearchForm.vue: 검색 조건 완성도 향상
- 검색 결과 정확도 개선

4️⃣ 엑셀 업로드 고도화

A. 업로드 규칙 엄격화
- 엑셀 템플릿 정의
- 필수 필드 검증
- 데이터 형식 검증 (IPA 코드, 언어 코드 등)

B. 오류 처리 개선
- 업로드 실패 시 상세 오류 메시지
- 부분 성공 시 결과 리포트

5️⃣ 기타 개선사항

A. 코드 테이블 정비
- tb_code, tb_code_grp 소리구분 코드 정리
- 다국어 지원 준비

B. UI/UX 개선
- 로딩 상태 표시
- 사용자 편의성 향상
